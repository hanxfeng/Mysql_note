+介绍：
	触发器是与表有关的数据对象，指在insert/update/delete之前或之后，触发并执行触发器中定义的SQL语句，可以协助应用在数据库端保持数据完整性、日志记录和数据校验等操作
	使用别名old和new引用触发器中发生变化的记录内容，与其他数据库相似，支持行级触发，不支持语句级触发，行级触发指每影响一行数据，触发器触发一次，语句级触发指每个语句执行时，不管影响几行数据，都只触发一次触发器
	可以通过new和old获取操作时的数据，具体获取范围如下表所示

| 触发器类型      | NEW           | OLD         |
| ---------- | ------------- | ----------- |
| insert型触发器 | 表示将要或者已经增加的数据 | 无           |
| update型触发器 | 表示将要或已经修改的数据  | 表示修改前的数据    |
| delete型触发器 | 无             | 将要或者已经修改的数据 |
语法：
	创建：
```
creat trigger 触发器名
before/after insert/update/delete on 表名 for each row
begin
	sql语句
end;
before/after指在执行前触发还是执行后触发
insert/update/delete指触发器类型
for each row指行级触发，目前MySQL仅支持行级触发
```
	查看：
		`show trigger;`
	删除：
		`drop tigger [数据库名] 触发器名;` 不指定数据库名则默认删除当前数据库的触发器
