事务的简介：

事务是一组操作的集合，它是一个不可分割的集合，事务会将所有操作作为一个整体向系统提交或撤销操作请求，这些操作要么同时成功要么同时失败。
具体流程是先开启事务，然后进行操作，临时修改数据，最后提交事务，数据更改。如果在进行操作时出现了错误，临时修改的数据就会回滚成原来的数据

事务的操作：

	更改事务提交方式：
	
		select @@autocommit；
		
		查询事务的提交方式，1为自动提交，0为手动提交
		
	set @@autocommit = 0；
		
		将事务提交方式设置为手动提交
		
	commit；
		
		设置为手动提交事务需要使用commit才会提交事务执行代码
		
	rollback；
		
		事务执行过程中出了问题就可以使用rollback回滚事务

	开启手动提交的事务
		
		使用语句：start transaction或begin
		
		执行后会开启一个手动提交的事务，直到使用commit提交为止
		
		提交与回滚的语句与上相同

事务的四大特性：

	原子性：事务是不可分割的最小操作单元，要么全部成功，要么全部失败
	
	一致性：事务完成时必须使所有数据都保持一致的状态
	
	隔离性：数据库系统提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行
	
	持久性：事务一旦提交或回滚，它对数据库中的数据影响是永久的

并发事务问题：

	脏读：
	
		一个事务读到另一个事务还未提交的数据，即第一个事务还在执行中未提交，此时正被第一个事务操作的事务被第二个事务读取
		
	不可重复读：
	
		一个事务中执行了两次查询操作，第一次查询操作后查询的数据被另一个事务改变，被改变的数据又被第一个事务第二次查询
	
	幻读：
	
		一个事务首先执行查询语句，此时显示数据不存在，然后执行插入语句，但在此之间，这个位置被另一个事务插入了数据，再次执行查询语句，发现依然没有数据，这就是幻读

事务隔离级别（用于解决并发事务问题）：

主要有四种：

|                                   |     |       |     |
| --------------------------------- | --- | ----- | --- |
| 隔离级别                              | 脏读  | 不可重复读 | 幻读  |
| read uncommitted（读，未提交）           | √   | √     | √   |
| read committed（读，已提交）             | x   | √     | √   |
| repeatable read（可重复读，mysql默认隔离级别） | x   | x     | √   |
| serializable（串行化）                 | x   | x     | x   |
| √代表会出现，x代表不会出现                    |     |       |     |

性能从上到下递减，安全性从上到下递增，√代表会出现该问题，x代表不会出现

查看事务的隔离级别：select @@transaction_isolation

设置事务隔离级别：set [session/global] transaction isolation level read uncommitted/read committed/repeatable read/serializable

session 代表仅对当前窗口有效，global代表对所有窗口有效
