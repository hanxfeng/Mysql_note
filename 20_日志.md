错误日志：
	记录了mysql启动和停止时，以及服务器在运行过程中发生任何严重错误时的相关信息。日志文件名为mysqld.log 查看相关配置：`show variables like %log_error%`
二进制日志：
	记录了所有的DDL语句和DML语句，但不包括查询语句（select,show）
	作用：
		1.灾难时的数据恢复
		2.MySQL主从复制
	查看相关配置：`show variables like %log_bin%`
	日志格式：
		statement：
			基于sql语句的日志记录，记录的是sql语句，对数据进行修改的sql都会记录在日志文件中
		row：
			基于行的日志记录，记录的是每一行的数据变更，默认的日志格式
		mixed：
			混合了以上两种格式，默认采用第一种，特殊情况下使用第二种
		查询日志格式：
			`show variables like %binlog_format%`
	查看日志：
```
		mysqlbinlog [参数列表] 日志文件名称
		参数选项：
			-d ：指定数据库名称，之列出指定数据库的相关操作
			-o ：忽略日志的前n行命令
			-v ：将行事件重构为sql语句
			-w ：将行事件重构为sql语句并输出注释信息
```
	日志删除：
```
		reset master:
			删除所有binlog日志，删除后日志编号将重新从000001开始
		purge master logs to '日志文件名' ：
			删除该日志编号前所有日志
		purge master logs before '时间' ：
			删除该时间之前的所有日志
		设置服务器配置，过期后自动删除日志文件:
			show variables like %binlog_expire_logs_seconds%
			单位是秒，默认30天
```
查询日志：
	记录了客户端的所有操作语句，相比于二进制日志，查询日志也会记录select和shwo语句等查询语句，默认情况下，查询日志是关闭的
	相关sql语句：
		`show variables like %general%`
		给出两个结果：
			general_log : 是否开启查询日志
			general_log_file : 查询日志保存位置
慢查询日志：
	记录了所有执行时间超过参数long_query_time设置值并且扫描记录数不小于min_examined_row_limit的所有sql语句的日志，默认不开启，long_query_time默认为10秒，最小为0，精度可达微秒
	开启参数： `slow_query_log`参数值为1
	默认情况下，不会记录管理语句，也不会记录不使用索引进行查找的查询，可以使用log_slow_admin_statement=1（记录执行较慢的管理语句）log_queries_not_using_indexes=1(记录执行较慢的未使用索引的语句)
