约束：约束在创建表时使用，位于数据类型后方，对表数据起限制作用

非空约束：not null

限制该字段值不能为null

唯一约束：unique

该字段的值都是唯一且不重复的，在使用该约束后向字段添加两个相同的值会出现报错

主键约束：primary key

主键是一行数据的唯一标识，要求非空且不重复，当同时与自增约束auto_increment同时使用时，即使添加值失败也会申请一个主键，导致自增，即会跳过创建失败的那个数，例如从3直接到5

默认约束：default

保存数据时如果未指定该字段的值则采用默认值，default 默认值

检查约束：check

保证字段值满足一个条件，条件可以是比较运算符等，用法：check 条件

外键约束

外键是指在一张表中，有一个字段，它的值是另一张表的主键，那么此时，这个字段就被称为外键

带有外键的表被称为子表，另一个表被称为父表

添加外键：

1.在创建表时添加：

需要添加在所有字段后

`Create table 表名（`

`字段名 数据类型`

`[constraint] [外键名称] foreign key (外键字段名) references 主表表名（主表主键名）`

其中外键名称可以随意取，外键字段名是要作为外键的字段的名称，主表列表名是主键名也是与子表中外键字段对应的字段

2.在创建表后添加：

alter table 表名 add constraint 外键名称 foreign key （外键字段名） references 主表表名（主表主键名）

其中表名是子表的表名，外键名称可以随意取，外键字段名是要作为外键的字段的名称，主表列表名是主键名也是与子表中外键字段对应的字段

删除外键：
alter table 表名 drop foreign key 外键名称

在父表中更新或删除行为 ：

当父表中更新或删除数据时，默认无法进行，因为有子表与其关联，但可以对其行为进行更改

alter table 表名 add constraint 外键名称 foreign key （外键字段名） references 主表（主表主键名） on update 行为 on delete 行为

on update指定的是更新时的行为，on delete指定的是删除时的行为

行为：

no action/restrict：

默认行为，当父表删除/更新记录时检查有无子表，如果有则不允许删除/更新

cascade

当父表删除/更新记录时检查有无子表，如果有则也删除/更新对应记录

set null

当父表删除/更新记录时检查有无子表，如果有则设该记录为null

set default

当父表删除/更新记录时检查有无子表，如果有则设置子表中该外键值为一个默认的值
