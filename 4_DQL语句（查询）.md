- # 基本查询：

	- ## 查询多个字段：
	
		- `select 字段1[as 别名1],字段2[as 别名2] from 表名;`
		
		- 在字段处写 * 为查询所有字段，在实际开发过程中尽量不要写*
		
		- 起别名时as可以省略，别名要加 ' '
	
	- ## 查询字段并去除重复记录：
	
		- `select distinct 字段列表 [as 别名列表] from 表名;`
		
		- 列表意为多个以逗号分隔的字段名等,不需要加【】或（），使用“，”分隔即可


- # 条件查询:

	- `Select 字段列表 from 表名 where 条件列表`
	
	- 条件列表中格式为： 字段名1 条件1 逻辑运算符 字段名2 条件2


- # 逻辑运算符：

	- ## 比较运算符：以下均为where后的条件
	
		- ,<等省略，与py一致
		
		- <>,!=：不等于
		
		- between and ：在某个范围内，含边界值,`between * and *`,前面写较小值，后面写最大值
		
		- in:后跟一个列表，符合列表中的数据都会被查询
		
			- 例：`select * from 表名 where 字段名 in (10,15,20);`
			
			- 在字段中所有值为10，15，20的数据都会被查询
		
		- like：模糊匹配，有_和%两种， `_`代表单个字符，%代表任意个字符
		
			- 进行模糊查询时要加引号，`__`为查询所有长度为2的数据
			
			- `%z`为查询所有结尾为z的数据 `z%`为查询所有开头为z的数据
			
			- `%z%`为查询所有带z的数据，查询z不在开头结尾的数据需要先查询所有带z然后排除z在开头和结尾的数据
		
		- is null：查询值为null的数据
		
		- ## 子查询操作符1：
		
			- ### in :
			
				- 在指定的集合内逐个选择
			
			- ### not in：
			
				+ 不在指定的集合范围内
			
			- ### any/some：
			
				- 外层查询结果满足任意一个子查询结果即可
			
			- ### all：
			
				- 外层查询结果必须满足所有子查询结果，且可使用> all 这种语法使查询出的数据大于all中的所有数据
	
	- ## 逻辑运算符：
	
		- 与py一致，and=&&，or=||，not = ！

- # 聚合函数：

	- `select 聚合函数(字段名) from 表名 [where 条件列表];`
	
	- 字段名处可填 * 且只能填写一个字段名
	
	- 执行顺序：
	
		- 先执行from从指定表中读取数据，然后执行where语句筛选出符合条件的数据
		
		- 然后执行select语句，筛选出指定的字段并应用聚合函数
	
	- null值不参与聚合函数计算


- # 聚合函数：
	
	- count：计数
	
	- max：求最大值
	
	- min：求最小值
	
	- avg：求平均值
	
	- sum：求和


- # 分组查询：

	- `select 字段列表,聚合函数(字段名) from 表名 [where 条件列表] group by 分组字段名 [having 条件列表] order by 排序条件;`
	
		- 聚合函数要在整条代码中带有group by 时才可使用
		
		+ select处出现聚合函数后面必须带有group by
		
		- 用于select查询的字段名与用group by 分组的字段名应保持一致,
		
		- 聚合函数用于计数或求均值，可以不加，但不加只会返回所有不重复值但不确定数量等特征
		
		- where是在分组前进行筛选，having是在分组后进行筛选
		
		- where处的条件与having处条件不同主要在于where在查询前执行，having在查询后执行
		
		- 执行顺序：
		
			- 先执行from语句，读取表中所有数据，然后执行where语句，筛选出符合条件的数据
			
			- 接下来执行group by语句，根据字段名进行分组，字段中每一个不同的值会形成一个组
			
			+ 然后执行select语句，对groub by 分组后的数据执行聚合函数，然后执行having语句，对聚合后的数据进行筛选
			
			+ 最后执行order by语句对数据进行排序。order by视频没讲，知道即可


- # 排序查询：

	- `select 字段列表 from 表名 order by 字段1 [排序方式1],字段2 [排序方式2];`
	
	- 排序方式有两种，一种是asc升序，另一种是desc降序,多字段指第一字段值相同则按第二字段排序


- # 分页查询：

	- `Select 字段列表 from 表名 limit 起始索引,查询记录数;`
	
		- 这里的起始索引是指从哪里开始查询
		
		- 起始索引为查询页数-1*每页数据数量
		
		- 查询记录数是指每一页的数据数量
		
		- 数据库不同分页查询函数不同
		
		- 也可不指定起始索引，此时效果可视为查询前n条记录。
	
	- ## 执行顺序：
	
		+ DQL的编写顺序与执行顺序不同，编写顺序为：
		
			+ `Select→from→where→group by→having→order by→limit`
		
		- 执行顺序为：
		
			- `from→where→group by→having→select→order by→limit`
