多表关系：

表结构通常有三种关系：

一对多（多对一）：

实现：在多的一方建立外键，对应一的一方的主键

多对多：

建立一个中间表，其中有两个外键，分别对应两个主键

一对一：多用于单表拆分，将一张表的基础字段放在一张表，其余字段放在另一张表中

在任意一方加入外键，关联另一方的主键，并设置外键唯一

多表查询概述：

多表查询语法：select * from 表1，表2 where 表1.字段1=表二.字段二；

select * from 表1，表2这种查询方式会将输出表一和表二的笛卡尔积（即表1中的每一项与表二中的每一项一一对应），需要消除无效的部分，所以要加上where筛选，字段一和字段二即为两个表相同的字段

多表查询的分类：

连接查询：

内连接：

相当于查询两个表之间有交集的部分，包括隐式内连接与显式内连接
隐式内连接语法：
```
select 字段列表 from 表1,表2 where 条件（一般是表一外键等于表二主键）
```
显式内连接：
```
select 字段列表 表1 [inner] join 表2 on 连接条件 
```

隐式内连接与显式内连接至少语法不同，实质是相同的，需求相同时，隐式内连接的where条件与显式内连接的on条件是一样的，显式内连接的inner可以省略

在查询时可以通过`select * from 表1 表1别名 ，表2 表二别名;` 来为表起别名，当为表起了别名之后，这条sql语句中其他位置的表名也需要使用别名来代替，不然会报错

外连接：

左外连接与右外连接的连接条件一般都是一个表的外键等于另一个表的主键

左外连接：

```
select 字段列表 from 表1名 left [outer] join 表2名 on 条件
```

查询表一的所有数据和表二与表一交集部分的数据

右外连接：

```
select 字段列表 from 表1名 right [outer] join 表2名 on 条件
```

查询表二的所有数据和表二与表一交集部分的数据

自连接：

自连接可以认为是将一张表砍成两张表进行查询，举视频中的例子来说，就是表中有员工的姓名，员工的id,员工直属领导的id,但没有直属领导的姓名，此时就可以使用自连接来查询员工直属领导的姓名，自连接可以使用内连接的语法也可以使用外连接的语法，不过需要在from后起两个别名，然后在表1的地方使用别名1,表二的地方使用别名2

联合查询：

```
Select 字段列表 from 表A

Union [all]

Select 字段列表 from 表B
```

将两条查询结果合并起来，形成一个新的查询结果集

只是简单的将结果合并，当使用all时，会显示重复的查询结果，删掉后不显示重复的结果

两个字段列表中字段的数量需要一致，而且数据类型需要相同，可以合并同样是int的id和age,但不能合并一个是str的name和一个是int的age

子查询：

子查询是指在一个增删改查语句中嵌套一个select查询语句，可以在from，where，select后
可以理解为将内层查询的结构作为条件输入到外层函数中，子查询部分要用括号括起来，可进行多层嵌套

我将外层的查询语句称为外层查询，子查询称为内层查询

标量子查询：子查询结果为单个值

	这个子查询的查询结果为单个值，常用>,<,=等比较运算符将子查询结果作为外层查询条件

列子查询：子查询结果为一列

	这个子查询的结果是一列多行，常用in,not in,some，any,all等子查询操作符 将子查询结果作为外层查询条件

行子查询：子查询结果为一行：

	行子查询应使用where（字段1，字段2……）=（子查询）的形式，常用比较运算符和自查询操作符

表子查询：子查询结果为多行多列

	主要使用 in 进行，使用where（字段1，字段2……）in（子查询）的形式

	也可以用在from后，作为一张表进行查询（此时应起一个别名方便操作）


可以将一个需求分成多个需求来理清子查询编写思路
